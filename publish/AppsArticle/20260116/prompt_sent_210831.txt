OCR手帳
は、紙のノートや手帳をカメラでスキャンして、デジタルメモとして保存できる革新的な手帳アプリです。OCR（光学文字認識）技術を活用し、撮影した画像から自動的にテキストを抽出。手書きのメモも、印刷された文字も、簡単にデジタル化できます。
✨ 主な機能
1. 📸 OCRスキャン機能
カメラスキャン
: アプリ内カメラで直接撮影してOCR処理
ライブラリから選択
: 既存の写真からもテキストを抽出可能
高精度認識
: Cloudflare OCR APIを使用した高精度な文字認識
日付自動抽出
: スキャンしたテキストから日付を自動検出
画像保存
: スキャンした画像をメモと一緒に保存

日付別整理
: カレンダー形式でメモを日付別に整理
リスト表示
: 今日のメモを一覧で確認
画像サムネイル
: メモリストに最新画像のサムネイルを表示
検索機能
: タグや日付でメモを簡単に検索
編集・削除
: いつでもメモの編集や削除が可能
3. 🎤 音声入力
リアルタイム音声認識
: 話した内容を即座にテキスト化
日本語・英語対応
: 多言語での音声入力に対応
既存テキストに追記
: 音声認識したテキストを既存のメモに追加可能
4. 📅 カレンダービュー
月間カレンダー
: 月単位でメモを視覚的に確認
日付タップ
: カレンダーから日付をタップして直接メモを表示
メモ数表示
: 各日付にメモの数をバッジで表示
5. 🎨 カスタマイズ可能なUI
カラーテーマ

- bigBanner.tiles[0].imgSrc は、PAGE_ASSETS_JSON内の og_image があればそれを使う。なければ images[0].src を使う

- article.bodyHtml 内では、一次情報に含まれる画像URLやリンクURLを必要に応じて引用してよい（捏造禁止）

【要件】
- 全体<h1>＋<h2>×5章
- 章構成は、結論（アプリができること＋おすすめ使用シーン）→実際にダウンロードして使ってみた→おすすめユースケース→代替アプリとの比較→注意点と弱点
- 各章500文字以上
- 章ごとに役割と語り口を変える（同一構成禁止）
- 定型句・冗長な言い換え・無難な説明は禁止
- 「抽出した本文」とは違う順序で記事を構成すること
- 「抽出した本文」内にあるフレーズや言い回しは絶対に使用不可。
- 少なくとも1回は<table>を使う
- 各章の本文中に、必ず1行だけ強調文（<p class="em">〜</p>）を挿入する。

【単調さ対策（追加ルール）】
- 章の本文冒頭に「短い1文の導入」を置いてよい（説明調ではなく、気づき/違和感/一言コメントにする）。
- 箇条書き（ul/ol/li）は使用禁止。代わりに短い1文だけの<p>を2〜4行連続で置いてよい。
- 中見出し（h3等）は使用禁止。
- Before/Afterを必ずボックス化せよ（1章につき最大1セット）。使用HTMLは固定：
  <div class="soft-box"><strong>Before：</strong>...</div>
  <div class="soft-box"><strong>After：</strong>...</div>
- <table>は料金比較以外にも「利用シーン×効果」「場面×変化」などの整理用途で使ってよい。tableは<thead><tbody>を必ず含める。
