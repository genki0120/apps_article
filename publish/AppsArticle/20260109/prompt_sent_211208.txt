NijiCAD for Windows
NijiCAD is a lightweight, modern 2D CAD application available for free. Based on the concept of "Focus on Drawing," it provides a versatile drawing environment not limited to specific industries.
The user interface and command line support multiple languages, allowing users around the world to operate intuitively in their native languages.
Key Features
8 Language Support
: Supports Japanese, English, French, German, Spanish, Chinese (Simplified), Chinese (Traditional), and Korean.
Intuitive Operability
: Combines command-line operation for those accustomed to traditional CAD with a graphical UI for beginners.
High Compatibility
: Supports DXF input/output and JWW input, allowing smooth utilization of existing drawing assets.
Powerful Drawing Functions
: Covers essential functions for practical work such as layer management, dimensioning, hatching, and polylines.
It is a versatile and efficient CAD tool that meets all 2D drawing needs, such as drawings, technical sketches, and diagrams.

- bigBanner.tiles[0].imgSrc は、PAGE_ASSETS_JSON内の og_image があればそれを使う。なければ images[0].src を使う
- bigBanner.tiles[0].linkUrl は、PAGE_ASSETS_JSON内の product_links[0].url があればそれを使う。なければ参照URL https://izanami.dev/post/2ae37a79-5af9-47a0-b854-8deab09d3217 にする
- article.bodyHtml 内では、一次情報に含まれる画像URLやリンクURLを必要に応じて引用してよい（捏造禁止）
PAGE_ASSETS_JSON の images が存在する場合、本文冒頭（lead直後など）に <img> を1枚だけ入れてよい。なるべくアプリ名に紐づくアイキャッチ画像かロゴの画像を反映すること（srcは assets から、altは不明なら短く一般的にしたのち画像の下に表示）。

【要件】
- 全体<h1>＋<h2>×5章
- 各章500文字以上
- 章ごとに役割と語り口を変える（同一構成禁止）
- 体験のBefore/Afterを軸に、友達にすすめる口調で書く
- 定型句・冗長な言い換え・無難な説明は禁止
- 利用シーン章では「定番→意外→さらに意外」の順
- 少なくとも1回は<table>を使う
- 最終章までに使用までの3ステップ例を記述すること
- 各章の本文中に、必ず1行だけ強調文（<p class="em">〜</p>）を挿入する。
- 「抽出した本文」内にある以下の文章は無視してください。

【単調さ対策（追加ルール）】
- 章の本文冒頭に「短い1文の導入」を置いてよい（説明調ではなく、気づき/違和感/一言コメントにする）。
- 箇条書き（ul/ol/li）は使用禁止。代わりに短い1文だけの<p>を2〜4行連続で置いてよい（1章につき最大1回）。
- 中見出し（h3等）は使用禁止。
- Before/Afterを必ずボックス化せよ（1章につき最大1セット）。使用HTMLは固定：
  <div class="soft-box"><strong>Before：</strong>...</div>
  <div class="soft-box"><strong>After：</strong>...</div>
- <table>は料金比較以外にも「利用シーン×効果」「場面×変化」などの整理用途で使ってよい。tableは<thead><tbody>を必ず含める。
